# PRD 文件 — 晚餐輪盤系統

## 1. 專案概述
- **背景與動機**：決定晚餐吃什麼常常是一個令人選擇障礙的難題。為了解決這個每天發生的痛點，建立一個「晚餐輪盤系統」，讓使用者可以透過富有趣味性的方式隨機決定晚餐。
- **目標用戶**：我（以及任何有選擇困難的朋友）。
- **核心價值主張**：用簡單、有趣、無壓力的轉盤方式，一秒解決「晚餐吃什麼」的世紀難題，並具備自訂選項的彈性。

## 2. 功能需求
以下是系統的主要功能與對應的使用者故事：

1. **新增/管理晚餐品項**
   - 作為使用者，我希望可以新增或刪除晚餐選項（如：麥當勞、巷口麵店），以便讓轉盤上的選項符合我目前的喜好與清單。
2. **隨機產生品項（轉盤功能）**
   - 作為使用者，我希望可以點擊一個按鈕後，系統用「轉盤」的方式動畫旋轉並隨機決定一個晚餐選項，以便我能充滿期待地獲得結果。
3. **查看過往歷史紀錄**
   - 作為使用者，我希望能夠看到最近幾次轉盤抽中的晚餐紀錄，以便我知道最近吃了什麼，避免重複吃太膩。
4. **晚餐品項權重/機率設定（選配）**
   - 作為使用者，我希望可以為特別喜歡的餐點設定較高機率（佔轉盤較大面積），以便它們更容易被抽中。
5. **啟用/停用部分品項**
   - 作為使用者，我希望可以暫時關閉某些不想吃的選項（不直接刪除），以便今天的轉盤不會抽到它們。

## 3. 非功能需求
- **技術限制**：
  - 後端：Flask (Python)
  - 前端渲染：Jinja2 模板引擎
  - 資料庫：SQLite，並使用 SQLAlchemy 或原生 SQL 管理資料
  - 前端特效：需使用 Vanilla JavaScript 與 CSS 實作轉盤動畫，打造流暢且高品質的視覺體驗。
- **效能與安全考量**：
  - 網頁需具備 RWD（響應式設計），讓我在手機上也能輕鬆點擊轉盤。
  - 對表單輸入進行基礎的跳脫（Escape）處理，防止 XSS 攻擊。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須具備)**：
  - 晚餐清單的 CRUD（新增、讀取、刪除）。
  - 一個能隨機抽出晚餐的輪盤介面。
  - 資料能永久儲存於 SQLite 資料庫。
- **Should Have (應該具備)**：
  - 轉盤的旋轉動畫（而不只是直接跳出結果），增強互動感。
  - 手機瀏覽友善的介面佈局。
- **Nice to Have (有會更好)**：
  - 歷史紀錄追蹤。
  - 暫時啟用/停用特定晚餐選項。
